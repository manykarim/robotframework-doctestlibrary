On the Reliability of Visual Tests

A long typographic page: full-page capture, dense font rendering, sub-pixel anti-aliasing.

The problem

Rendering the same glyphs twice rarely produces identical pixels across engines. Hinting, kerning and gamma differ; humans see the same page, pixel comparison sees thousands of changes.

A reliable visual test distinguishes what changed for the user from what changed for the rasterizer.

The approach

Compare structure with structure and pixels with budgets: semantic DOM snapshots answer did content change?, edge classification answers is this just anti-aliasing?, and everything else deserves a human—or a well-prompted model—in the review loop.

Coda

Ship baselines with the code, review diffs like code, and let the noise pass through the filters built for it.
